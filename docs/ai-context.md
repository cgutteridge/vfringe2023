# AI Context

## What this repository is

This directory is a WordPress plugin used to power Ventnor Fringe festival pages with custom event views.

It is not only a downloader. The `download-boxoffice/` subdirectory is a separate Node-based helper that produces one input file for the plugin.

## Git workflow

One maintainer; no peer review or pull requests. New work goes on a branch, is tested there, then merges to `main` and is pushed.

## Main runtime flow

### 1. Bootstrap

`chrisvf.php` loads the feature modules:

- `itinerary.php`
- `map.php`
- `grid.php`
- `ff.php`
- `roulette.php`
- `now_and_next.php`
- `montydump.php`
- `search.php`
- `byday.php`
- `mobile.php`
- `grid-print.php`
- `itinerary-ics.php`

### 2. Shared event build

`chrisvf_get_info()` memoizes the merged dataset in memory for the current request and can also write a JSON cache file under `/tmp`.

`chrisvf_wp_events()`:

- loads WordPress `ajde_events` posts
- extracts event metadata, locations, categories, and thumbnails
- expands repeats into per-occurrence items
- merges rows from `boxoffice-events.tsv` and `extras.tsv`

### 3. Feature rendering

Each shortcode file renders from the shared event array:

- `chrisvf_itinerary`, `chrisvf_saved_itinerary`, `chrisvf_itinerary_slug`
- `chrisvf_map`
- `chrisvf_grid`
- `chrisvf_ff_alpha`, `chrisvf_ff_cat`, `chrisvf_ff_day`
- `chrisvf_now_and_next`
- `chrisvf_random`
- `chrisvf_search`
- `chrisvf_by_day`
- `chrisvf_montydump`

`mobile.php` also exposes non-shortcode routes:

- `/m` — standalone mobile programme shell
- `/m/json` — normalized JSON for that shell

`itinerary-ics.php` exposes:

- `/itinerary-ics?ids=` — downloadable multi-event calendar file for a saved itinerary

## External dependencies and assumptions

- WordPress
- EventON-style event data:
  - post type `ajde_events`
  - taxonomy `event_location`
  - taxonomy `event_type`
  - option `evo_tax_meta`
- The Events Calendar functions in itinerary-related rendering
- Local JSON/TSV assets kept in this repository
- Leaflet JS/CSS bundled locally in the plugin

## Local data files that matter

- `boxoffice-events.tsv`: imported ticketed events, usually generated by `download-boxoffice/`
- `extras.tsv`: manually curated additions
- `places.json`: map locations and venue-to-POI mapping
- `outlines.json`: map polygons
- `lines.json`: optional map lines, currently disabled in `map.php`

## Common failure modes

- Venue name drift between TSV files, WordPress terms, and `places.json`
- Changed TSV headings without corresponding parser changes
- Hard-coded year values becoming stale
- Absolute site paths no longer matching live page slugs
- Edits to event field names breaking downstream renderers
